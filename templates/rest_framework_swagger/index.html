Welcome to the documentation for the Wocat API. Please refer to the qcat docs in case of questions regrading the structure of the returned objects. The API is built on top of the django REST framework.

Requests to any resource requires an authentication token in the header of the request. The token can be obtained with this API (see: auth-token), a wocat user account is required to obtain a token. Pass the token in the request header as follows: 

Authorization: Token <your token here>

Example request: 
curl -H "Authentication: Token 5289bc42fb2e9322b3aa6238669734bacdc1ae60" http://qcat.wocat.net/en/api/v1/questionnaires/

The current version of the API is: 1.0. Docs for the older versions will be listed here as well.
